EN

This challenge requires you to manage a two-dimensional integer matrix entirely using raw pointers and dynamic memory allocation. You are tasked with implementing a set of functions that manipulate this matrix, specifically focusing on its dynamic resizing capabilities and element rearrangement, all while adhering to strict memory management principles. The problem explicitly forbids the use of `struct`, `typedef`, or file I/O, forcing a deeper reliance on pointer arithmetic and indirection.

Your primary goal is to implement three core functions. First, `addRow` will dynamically insert a new row into the matrix at a specified index. This operation involves reallocating the array of row pointers, shifting existing row pointers to accommodate the new row, and then allocating memory for the new row itself. Care must be taken to handle potential memory allocation failures and to ensure all existing data remains intact. Second, `removeColumn` will dynamically delete a column from the matrix at a given index. This requires iterating through each row, reallocating each row to reduce its size, and shifting elements within each row. This is particularly challenging as it affects every single row's memory block. Finally, `swapElements` should exchange the values of two arbitrary cells within the matrix, specified by their row and column indices.

Throughout the implementation, meticulous memory management is paramount. Every `malloc` and `realloc` call must be paired with a corresponding `free` operation when memory is no longer needed. Failure to properly manage memory, such as introducing memory leaks or attempting to access freed memory, will result in incorrect behavior. The problem tests your understanding of `pointer to pointer` concepts, dynamic array resizing, and robust error handling in a memory-constrained environment, pushing the boundaries of C's low-level memory control.

TR

Bu problem, tamamen ham işaretçiler ve dinamik bellek tahsisi kullanarak iki boyutlu bir tamsayı matrisini yönetmenizi gerektirir. Özel olarak, bu matrisin dinamik olarak yeniden boyutlandırılmasına ve elemanlarının yeniden düzenlenmesine odaklanan bir dizi fonksiyonu uygulamanız istenmektedir, tüm bunları sıkı bellek yönetimi prensiplerine bağlı kalarak yapmanız gerekmektedir. Problem, `struct`, `typedef` veya dosya G/Ç kullanımını kesinlikle yasaklar; bu da sizi işaretçi aritmetiği ve indirection'a daha derinlemesine güvenmeye zorlar.

Temel hedefiniz üç ana fonksiyonu uygulamaktır. İlk olarak, `addRow` fonksiyonu belirtilen bir indekse dinamik olarak yeni bir satır ekleyecektir. Bu işlem, satır işaretçileri dizisini yeniden tahsis etmeyi, mevcut satır işaretçilerini yeni satıra yer açmak için kaydırmayı ve ardından yeni satırın kendisi için bellek tahsis etmeyi içerir. Olası bellek tahsis hatalarını ele almaya ve tüm mevcut verilerin sağlam kalmasını sağlamaya özen gösterilmelidir. İkinci olarak, `removeColumn` fonksiyonu belirli bir indeksteki bir sütunu dinamik olarak silecektir. Bu, her bir satırı yinelemeyi, her satırı boyutunu küçültmek için yeniden tahsis etmeyi ve her satırdaki elemanları kaydırmayı gerektirir. Bu, her bir satırın bellek bloğunu etkilediği için özellikle zordur. Son olarak, `swapElements` matristeki iki rastgele hücrenin değerlerini, satır ve sütun indeksleri ile belirtildiği şekilde değiştirmelidir.

Uygulama boyunca, titiz bellek yönetimi kritik öneme sahiptir. Her `malloc` ve `realloc` çağrısı, belleğe artık ihtiyaç kalmadığında ilgili bir `free` işlemiyle eşleştirilmelidir. Belleği düzgün bir şekilde yönetmemek, örneğin bellek sızıntıları oluşturmak veya serbest bırakılmış belleğe erişmeye çalışmak, yanlış davranışa yol açacaktır. Bu problem, işaretçinin işaretçisi kavramlarını, dinamik dizi yeniden boyutlandırmayı ve bellek kısıtlı bir ortamda sağlam hata işlemeyi anlamanızı test ederek, C'nin düşük seviyeli bellek kontrolünün sınırlarını zorlamaktadır.
