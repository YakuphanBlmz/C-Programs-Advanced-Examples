EN

In C programming, managing memory for multi-dimensional arrays, especially when their dimensions are not fixed at compile time or can change during runtime, is a significant challenge. This problem focuses on mastering dynamic memory allocation, pointer arithmetic, and `realloc` in a complex scenario involving a 'jagged matrix'. A jagged matrix is a 2D array where each row can have a different number of columns. This setup requires careful handling of `int**` (a pointer to an array of pointers to integers) where each `int*` points to a dynamically allocated row.

The task is to implement a function, `insertColumn`, which dynamically inserts a new column at a specified index across all rows of an existing jagged matrix. This operation means that for every row in the matrix, its length must increase by one, and existing elements from the insertion point onwards must be shifted to the right to accommodate the new value. The new elements in the inserted column should be initialized to a given default value. You are explicitly forbidden from using `struct`, `typedef`, or any file I/O operations, making this problem a pure test of raw pointer and dynamic memory manipulation.

Your `insertColumn` function should have the signature `void insertColumn(int** matrix, int* row_lengths, int num_rows, int col_index_to_insert, int default_value)`. The `matrix` parameter points to an array of `int*`, where each `int*` points to a row. The `row_lengths` parameter is an array indicating the current length of each corresponding row; this array *must be updated* by your function to reflect the new lengths. `num_rows` indicates the total number of rows. `col_index_to_insert` specifies the 0-based index where the new column should be added, and `default_value` is the integer to be placed in the newly inserted column. Proper error handling for `realloc` (e.g., checking for `NULL` returns) is expected, although for simplicity in a competitive context, an `exit(EXIT_FAILURE)` might be acceptable if `realloc` fails. Your `main` function should demonstrate the allocation, initialization, manipulation, and correct deallocation of such a jagged matrix.

C programlamada, çok boyutlu diziler için bellek yönetimi, özellikle boyutları derleme zamanında sabitlenmediğinde veya çalışma zamanında değişebildiğinde önemli bir zorluk teşkil eder. Bu problem, 'düzensiz matris' (jagged matrix) içeren karmaşık bir senaryoda dinamik bellek tahsisini, işaretçi aritmetiğini ve `realloc` kullanımını ustalaşmaya odaklanmaktadır. Düzensiz matris, her satırın farklı sayıda sütuna sahip olabileceği 2 boyutlu bir dizidir. Bu yapı, `int**` (işaretçilere işaret eden bir işaretçi dizisi) ile dikkatli bir işlem gerektirir, burada her `int*` dinamik olarak tahsis edilmiş bir satırı işaret eder.

Görev, mevcut düzensiz bir matrisin tüm satırlarında belirtilen bir indekse dinamik olarak yeni bir sütun ekleyen `insertColumn` adlı bir fonksiyon uygulamaktır. Bu işlem, matristeki her satır için uzunluğunun bir artması gerektiği ve ekleme noktasından sonraki mevcut elemanların yeni değeri barındırmak için sağa kaydırılması gerektiği anlamına gelir. Eklenen sütundaki yeni elemanlar, verilen varsayılan bir değerle başlatılmalıdır. Bu problemi saf işaretçi ve dinamik bellek manipülasyonu testi haline getirmek için `struct`, `typedef` veya herhangi bir dosya G/Ç işlemi kullanmanız kesinlikle yasaktır.

`insertColumn` fonksiyonunuzun imzası `void insertColumn(int** matrix, int* row_lengths, int num_rows, int col_index_to_insert, int default_value)` şeklinde olmalıdır. `matrix` parametresi, her `int*`'in bir satırı işaret ettiği bir `int*` dizisine işaret eder. `row_lengths` parametresi, her ilgili satırın mevcut uzunluğunu gösteren bir dizidir; bu dizi, yeni uzunlukları yansıtacak şekilde fonksiyonunuz tarafından *güncellenmelidir*. `num_rows`, toplam satır sayısını gösterir. `col_index_to_insert` spesifik olarak yeni sütunun eklenmesi gereken 0 tabanlı indeksi belirtir ve `default_value`, yeni eklenen sütuna yerleştirilecek tam sayıdır. `realloc` için uygun hata işleme (örneğin, `NULL` dönüşlerini kontrol etme) beklenmektedir, ancak rekabetçi bir bağlamda basitleştirmek için `realloc` başarısız olursa `exit(EXIT_FAILURE)` kabul edilebilir. `main` fonksiyonunuz, böyle bir düzensiz matrisin tahsisini, başlatılmasını, manipülasyonunu ve doğru şekilde serbest bırakılmasını göstermelidir.
