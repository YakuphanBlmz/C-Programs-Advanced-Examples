EN

As a C programmer, you are tasked with implementing a highly dynamic data structure without using `struct` or `typedef`. Your goal is to manage a collection of integer sets, where both the number of sets in the collection and the number of elements within each individual set can change dynamically during runtime. This structure effectively forms a 'jagged array' of integers, where each 'row' (set) can have a different number of 'columns' (elements), and the number of 'rows' itself is not fixed.

You must represent this collection using a pointer to a pointer (`int**`) to hold the dynamic array of sets, and a parallel dynamic array (`int*`) to keep track of the current number of elements in each respective set. The problem requires you to perform several complex memory management operations: dynamically adding new, empty sets to the collection, adding elements to existing sets (which may involve resizing individual set arrays), removing specific elements from a set (requiring element shifting and resizing), and completely removing an entire set from the collection (requiring freeing the set's memory, shifting the remaining set pointers and counts, and resizing the main collection arrays). Finally, a thorough memory cleanup is essential to prevent any leaks.

This challenge emphasizes meticulous use of `malloc`, `realloc`, and `free` across multiple levels of indirection. You must carefully manage memory for both the outer array of set pointers and each individual inner set array. Pay close attention to boundary conditions, potential `NULL` returns from `realloc` (although for this problem, assume successful allocations for simplicity), and ensuring all allocated memory is properly freed at the end. The solution should demonstrate a sequence of these operations to showcase the full dynamic capabilities.

TR

Bir C programcısı olarak, `struct` veya `typedef` kullanmadan oldukça dinamik bir veri yapısı uygulamanız istenmektedir. Amacınız, hem koleksiyondaki setlerin sayısı hem de her bir set içindeki eleman sayısı çalışma zamanında dinamik olarak değişebilen bir tamsayı setleri koleksiyonunu yönetmektir. Bu yapı, her 'satırın' (setin) farklı sayıda 'sütuna' (elemana) sahip olabildiği ve 'satır' sayısının da sabit olmadığı, etkili bir şekilde bir 'düzensiz dizi' (jagged array) oluşturur.

Bu koleksiyonu, setlerin dinamik dizisini tutmak için bir işaretçinin işaretçisi (`int**`) ve her bir setin mevcut eleman sayısını takip etmek için paralel dinamik bir dizi (`int*`) kullanarak temsil etmelisiniz. Problem, çeşitli karmaşık bellek yönetimi işlemleri yapmanızı gerektirir: koleksiyona yeni, boş setler dinamik olarak eklemek; mevcut setlere eleman eklemek (bireysel set dizilerini yeniden boyutlandırmayı gerektirebilir); bir setten belirli elemanları kaldırmak (eleman kaydırma ve yeniden boyutlandırma gerektiren); ve koleksiyondan tüm bir seti tamamen kaldırmak (setin belleğini serbest bırakmayı, kalan set işaretçilerini ve sayılarını kaydırmayı ve ana koleksiyon dizilerini yeniden boyutlandırmayı gerektiren). Son olarak, herhangi bir bellek sızıntısını önlemek için kapsamlı bir bellek temizliği zorunludur.

Bu zorluk, birden çok dolaylılık seviyesinde `malloc`, `realloc` ve `free`'nin titiz kullanımını vurgular. Hem set işaretçilerinin dış dizisi hem de her bir iç set dizisi için belleği dikkatlice yönetmelisiniz. Sınır koşullarına, `realloc`'tan gelebilecek potansiyel `NULL` dönüşlerine (ancak bu problem için, basitlik adına başarılı tahsisatları varsayalım) ve sonunda tahsis edilen tüm belleğin düzgün bir şekilde serbest bırakıldığından emin olmaya özellikle dikkat edin. Çözüm, tüm dinamik yetenekleri sergilemek için bu işlemlerin bir sırasını göstermelidir.
