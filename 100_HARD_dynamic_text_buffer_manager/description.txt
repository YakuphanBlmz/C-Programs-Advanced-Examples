EN

In C programming, managing dynamic collections of data without the aid of high-level abstractions like structs or complex library functions presents a significant challenge, especially when dealing with variable-length elements. This problem requires you to implement a basic dynamic text buffer system that handles a collection of text lines. Each line is a dynamically allocated string, and the collection itself is a dynamic array of pointers to these strings.

Your task is to manage a `char**` array, which serves as the primary buffer for storing pointers to individual text lines. Each `char*` within this array will point to a dynamically allocated C-style string. The system must support fundamental operations: adding new lines of text, modifying existing lines (which might necessitate resizing the memory allocated for that specific line), and deleting lines. Crucially, the main `char**` buffer itself must be dynamically resizable using `realloc` as lines are added or removed, optimizing memory usage and adapting to the changing number of lines.

This exercise rigorously tests your understanding of pointer arithmetic, dynamic memory allocation (`malloc`, `realloc`, `free`), and defensive programming. You must ensure that all memory allocations are checked for failure, all dynamically allocated memory is properly freed to prevent leaks, and operations like deleting a line correctly shift subsequent pointers and resize the main buffer. The absence of `struct` or `typedef` means you'll be working directly with raw pointers and primitive types, requiring meticulous manual memory management and index tracking. Pay close attention to reallocating strategies for both individual lines and the main line buffer to handle various scenarios, including shrinking the buffer when many lines are removed.

TR

C programlamada, `struct` gibi üst düzey soyutlamalar veya karmaşık kütüphane işlevleri kullanmadan dinamik veri koleksiyonlarını yönetmek, özellikle değişken uzunluktaki öğelerle uğraşırken önemli bir zorluk teşkil eder. Bu problem, metin satırları koleksiyonunu işleyen temel bir dinamik metin arabellek sistemi uygulamanızı gerektirir. Her satır dinamik olarak ayrılmış bir dizidir ve koleksiyonun kendisi, bu dizelere işaretçilerden oluşan dinamik bir dizidir.

Göreviniz, tek tek metin satırlarına işaretçileri depolamak için birincil arabellek görevi gören bir `char**` dizisini yönetmektir. Bu dizideki her `char*`, dinamik olarak ayrılmış bir C stili dizeye işaret edecektir. Sistem temel işlemleri desteklemelidir: yeni metin satırları ekleme, mevcut satırları değiştirme (bu, o belirli satır için ayrılan belleğin yeniden boyutlandırılmasını gerektirebilir) ve satırları silme. En önemlisi, ana `char**` arabelleğinin kendisi, satırlar eklendikçe veya kaldırıldıkça `realloc` kullanılarak dinamik olarak yeniden boyutlandırılabilmeli, böylece bellek kullanımını optimize etmeli ve değişen satır sayısına uyum sağlamalıdır.

Bu alıştırma, işaretçi aritmetiği, dinamik bellek tahsisi (`malloc`, `realloc`, `free`) ve savunmacı programlama anlayışınızı titizlikle test eder. Tüm bellek tahsislerinin başarısızlık için kontrol edildiğinden, tüm dinamik olarak ayrılan belleğin sızıntıları önlemek için düzgün bir şekilde boşaltıldığından ve bir satırı silme gibi işlemlerin sonraki işaretçileri doğru şekilde kaydırdığından ve ana arabelleği yeniden boyutlandırdığından emin olmalısınız. `struct` veya `typedef`'in olmaması, doğrudan ham işaretçiler ve ilkel türlerle çalışacağınız anlamına gelir; bu da titiz manuel bellek yönetimi ve dizin takibi gerektirir. Bireysel satırlar ve ana satır arabelleği için yeniden boyutlandırma stratejilerine, birçok satır kaldırıldığında arabelleği küçültmek de dahil olmak üzere çeşitli senaryoları ele almak için dikkatlice odaklanın.
