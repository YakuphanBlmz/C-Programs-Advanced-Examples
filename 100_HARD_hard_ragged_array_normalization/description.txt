EN

This problem challenges your mastery of pointers and dynamic memory management in C by tasking you with handling "ragged" or "jagged" 2D arrays. Unlike traditional 2D arrays where all rows have the same length, a ragged array is an array of pointers, where each pointer points to a dynamically allocated array (a "row") that can have a different number of elements. This structure requires careful and explicit memory management for each individual row and for the array of row pointers itself.

Your primary task is to implement a function that "normalizes" such a ragged 2D array. Normalization, in this context, means resizing every row in the array so that all rows ultimately share the same length. This target length must be determined by finding the maximum length among all existing rows in the ragged array. If a row needs to be expanded to reach this maximum length, the newly allocated cells must be initialized with a specific `fillValue` (e.g., 0). This process necessitates the use of `realloc` for individual rows and meticulous tracking of their current sizes.

You are required to implement three functions: one to dynamically create such a ragged array and its associated length tracker, one to perform the normalization and potentially resize rows using `realloc`, and one to correctly deallocate all memory. The creation function will take the number of rows and an array specifying the initial length for each row, returning the `int**` array and updating a pointer to an array of integers that keeps track of the current length of each row. The normalization function must robustly handle `realloc` failures by returning an error code. Crucially, your solution must **not** use `struct`s, `typedef`s, or file I/O operations, relying purely on raw pointers and dynamic memory functions.

TR

Bu problem, C programlamada işaretçiler ve dinamik bellek yönetimi konusundaki ustalığınızı "düzensiz" veya "jagged" 2D dizilerle sınamaktadır. Geleneksel 2D dizilerin aksine, her satırın aynı uzunlukta olduğu düzensiz bir dizi, her bir işaretçinin farklı sayıda öğeye sahip dinamik olarak tahsis edilmiş bir diziye ("satıra") işaret ettiği bir işaretçiler dizisidir. Bu yapı, her bir satır ve satır işaretçileri dizisinin kendisi için dikkatli ve açık bellek yönetimi gerektirir.

Temel göreviniz, böyle bir düzensiz 2D diziyi "normalleştiren" bir fonksiyonu uygulamaktır. Bu bağlamda normalleştirme, dizideki her satırı yeniden boyutlandırarak tüm satırların nihayetinde aynı uzunluğa sahip olmasını sağlamaktır. Bu hedef uzunluk, düzensiz dizideki tüm mevcut satırlar arasındaki maksimum uzunluk bulunarak belirlenmelidir. Bir satırın bu maksimum uzunluğa ulaşmak için genişletilmesi gerekiyorsa, yeni tahsis edilen hücreler belirli bir `fillValue` (örneğin 0) ile başlatılmalıdır. Bu süreç, bireysel satırlar için `realloc` kullanımını ve bunların mevcut boyutlarının titizlikle takip edilmesini gerektirir.

Üç fonksiyon uygulamanız gerekmektedir: biri düzensiz bir dizi ve ilgili uzunluk takipçisini dinamik olarak oluşturmak için, biri normalleştirme işlemini gerçekleştirmek ve potansiyel olarak `realloc` kullanarak satırları yeniden boyutlandırmak için, ve biri tüm belleği doğru şekilde serbest bırakmak için. Oluşturma fonksiyonu, satır sayısını ve her satırın başlangıç uzunluğunu belirten bir dizi alacak, `int**` diziyi döndürecek ve her satırın mevcut uzunluğunu takip eden bir tamsayı dizisine işaretçiyi güncelleyecektir. Normalleştirme fonksiyonu, `realloc` hatalarını bir hata kodu döndürerek sağlam bir şekilde ele almalıdır. Önemli olarak, çözümünüz `struct`, `typedef` veya dosya G/Ç işlemlerini **kullanmamalı**, yalnızca ham işaretçilere ve dinamik bellek fonksiyonlarına dayanmalıdır.
