EN

In the realm of advanced C programming, mastering dynamic memory allocation and pointer arithmetic is paramount. This problem challenges your understanding by requiring you to manage complex data structures without the aid of higher-level abstractions like `struct` or `typedef`. You are tasked with processing sensor data, represented as a two-dimensional dynamic array of integers (a matrix), where each row corresponds to a sensor array's readings and each column an individual sensor value.

Your primary objective is to implement a mechanism to filter these sensor arrays. Specifically, you need to create a *new* dynamically allocated matrix that contains only the "active" sensor arrays. An array is considered active if the sum of its readings (elements in a row) exceeds a predefined threshold. The number of columns in the filtered matrix must remain identical to the original, but the number of rows will vary based on how many sensor arrays meet the activity criterion. This task demands meticulous use of `malloc`, `free`, and careful manipulation of `int**` pointers.

The solution should encompass several critical steps. First, dynamically allocate an initial `R x C` matrix and populate it with arbitrary integer values. Next, iterate through this matrix, calculating the sum for each row. Based on these sums and the given threshold, determine which rows are to be included in the resultant matrix. Then, dynamically allocate the *new* matrix with the appropriate number of rows (and `C` columns), copying only the active rows from the original. Finally, and crucially, ensure all dynamically allocated memory, for both the original and the new matrix, is correctly deallocated using `free` to prevent memory leaks. The problem explicitly forbids the use of `struct`, `typedef`, and file I/O, pushing you to rely purely on pointers and dynamic memory primitives.

TR

İleri düzey C programlamasında, dinamik bellek yönetimi ve işaretçi aritmetiğinde ustalık büyük önem taşır. Bu problem, `struct` veya `typedef` gibi üst düzey soyutlamalara ihtiyaç duymadan karmaşık veri yapılarını yönetme yeteneğinizi test edecektir. Göreviniz, sensör verilerini işlemek; bu veriler, iki boyutlu dinamik bir tamsayı dizisi (bir matris) olarak temsil edilmektedir. Her satır bir sensör dizisinin okumalarını, her sütun ise tek bir sensör değerini ifade eder.

Temel hedefiniz, bu sensör dizilerini filtrelemek için bir mekanizma uygulamaktır. Özellikle, yalnızca "aktif" sensör dizilerini içeren *yeni*, dinamik olarak tahsis edilmiş bir matris oluşturmanız gerekmektedir. Bir dizi, okumalarının toplamı (bir satırdaki elemanların toplamı) önceden tanımlanmış bir eşiği aşıyorsa aktif kabul edilir. Filtrelenmiş matristeki sütun sayısı orijinaliyle aynı kalmalı, ancak satır sayısı kaç sensör dizisinin aktivite kriterini karşıladığına göre değişecektir. Bu görev, `malloc`, `free` ve `int**` işaretçilerinin dikkatli kullanımını gerektirir.

Çözüm, birkaç kritik adımı içermelidir. İlk olarak, başlangıçta `R x C` boyutlarında dinamik bir matris tahsis edin ve keyfi tamsayı değerleriyle doldurun. Ardından, bu matris üzerinde yineleyerek her satırın toplamını hesaplayın. Bu toplamlara ve verilen eşiğe dayanarak, hangi satırların sonuç matrisine dahil edileceğine karar verin. Daha sonra, uygun sayıda satıra (ve `C` sütuna) sahip *yeni* matrisi dinamik olarak tahsis edin ve orijinalden yalnızca aktif satırları kopyalayın. Son olarak ve en önemlisi, hem orijinal hem de yeni matris için dinamik olarak tahsis edilmiş tüm belleğin `free` kullanılarak doğru şekilde serbest bırakıldığından emin olun, böylece bellek sızıntıları önlenir. Problem, `struct`, `typedef` ve dosya G/Ç kullanımını açıkça yasaklayarak, tamamen işaretçilere ve dinamik bellek ilkellerine güvenmenizi gerektirir.
