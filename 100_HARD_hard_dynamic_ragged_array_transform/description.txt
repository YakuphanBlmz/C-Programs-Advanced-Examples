EN

You are tasked with managing a highly dynamic and irregularly structured collection of integer data in C. This problem requires extensive use of pointers and dynamic memory allocation (`malloc`, `free`) to create, manipulate, and deallocate a 'ragged' 2D array. A ragged array is a 2D array where each row can have a different number of columns. Crucially, you are forbidden from using `struct`s or `typedef`s to define custom data types or manage row metadata. Instead, the length of each row's data elements must be stored within the first element (`row[0]`) of that row's allocated memory block, with the actual data beginning from `row[1]`. This constraint forces a deeper understanding of raw pointer manipulation and memory layout.

The challenge involves three primary operations. First, you must implement a function `create_ragged_array` that dynamically allocates a ragged 2D array with a random number of rows and random, varying column lengths for each row, adhering to the `row[0]` length convention. The array elements should be initialized with a simple formula (e.g., based on row and column indices). This function must properly allocate an array of pointers, and then allocate separate memory blocks for each row, ensuring error handling for `malloc` failures.

Second, a `flatten_ragged_array` function is required, which takes such a ragged array and transforms it into a single, contiguous 1D dynamic array. This function must first calculate the total number of data elements across all rows (excluding the length indicators), then allocate a single block of memory for the 1D array, and finally copy all data elements sequentially into this new array. Third, you need to implement `expand_flattened_array`, which performs the reverse operation: taking the flattened 1D array and re-distributing its elements into a *new*, dynamically allocated ragged 2D array. This new array will have a different number of rows and potentially different row lengths than the original, again following the `row[0]` length convention for each new row. This requires careful calculation of new row distributions and allocations.

Throughout these operations, proper memory deallocation is critical. You must implement a `free_ragged_array` function that correctly releases all memory associated with a ragged array (each row, then the array of pointers itself) and ensure the flattened array is also freed. The problem emphasizes sophisticated pointer arithmetic, robust dynamic memory management, and careful handling of dynamic dimensions purely with pointers, without the aid of higher-level data structures like `struct`s or `typedef`s, nor any file I/O operations. Error handling for memory allocation failures (`malloc` returning `NULL`) should lead to program termination after cleaning up any already allocated memory.

TR

C programlama dilinde oldukça dinamik ve düzensiz yapılandırılmış bir tamsayı veri koleksiyonunu yönetmekle görevlendirildiniz. Bu problem, 'düzensiz' (ragged) bir 2D diziyi oluşturmak, işlemek ve bellekten serbest bırakmak için işaretçilerin ve dinamik bellek yönetiminin (`malloc`, `free`) yoğun kullanımını gerektirmektedir. Düzensiz bir dizi, her satırın farklı sayıda sütuna sahip olabileceği 2D bir dizidir. En önemlisi, özel veri türlerini tanımlamak veya satır meta verilerini yönetmek için `struct` veya `typedef` kullanmanız yasaktır. Bunun yerine, her satırın veri elemanlarının uzunluğu, o satırın ayrılmış bellek bloğunun ilk elemanında (`row[0]`) saklanmalı ve gerçek veriler `row[1]`'den başlamalıdır. Bu kısıtlama, ham işaretçi manipülasyonu ve bellek düzeni hakkında daha derin bir anlayış gerektirir.

Problem üç ana işlemi içermektedir. İlk olarak, rastgele sayıda satıra ve her satır için rastgele, değişen sütun uzunluklarına sahip düzensiz bir 2D diziyi dinamik olarak ayıran ve `row[0]` uzunluk kuralına uyan bir `create_ragged_array` fonksiyonu uygulamanız gerekmektedir. Dizi elemanları basit bir formülle (örneğin, satır ve sütun indekslerine göre) başlatılmalıdır. Bu fonksiyon, bir işaretçi dizisi ayırmalı ve ardından her satır için ayrı bellek blokları ayırarak `malloc` hataları için hata işlemeyi sağlamalıdır.

İkinci olarak, böyle bir düzensiz diziyi alıp tek, bitişik bir 1D dinamik diziye dönüştüren bir `flatten_ragged_array` fonksiyonu gereklidir. Bu fonksiyon, önce tüm satırlardaki toplam veri elemanı sayısını (uzunluk göstergeleri hariç) hesaplamalı, ardından 1D dizi için tek bir bellek bloğu ayırmalı ve son olarak tüm veri elemanlarını sırayla bu yeni diziye kopyalamalıdır. Üçüncü olarak, ters işlemi gerçekleştiren bir `expand_flattened_array` fonksiyonunu uygulamanız gerekmektedir: düzleştirilmiş 1D diziyi alıp elemanlarını *yeni*, dinamik olarak ayrılmış bir düzensiz 2D diziye yeniden dağıtmak. Bu yeni dizi, orijinalinden farklı sayıda satıra ve potansiyel olarak farklı satır uzunluklarına sahip olacak ve yine her yeni satır için `row[0]` uzunluk kuralına uyacaktır. Bu, yeni satır dağıtımlarının ve tahsislerinin dikkatli bir şekilde hesaplanmasını gerektirir.

Bu işlemler boyunca, düzgün bellek boşaltma kritik öneme sahiptir. Düzensiz bir diziyle ilişkili tüm belleği (her satır, ardından işaretçi dizisinin kendisi) doğru bir şekilde serbest bırakan bir `free_ragged_array` fonksiyonu uygulamanız ve düzleştirilmiş dizinin de serbest bırakıldığından emin olmanız gerekmektedir. Problem, gelişmiş işaretçi aritmetiğine, sağlam dinamik bellek yönetimine ve `struct` veya `typedef` gibi yüksek seviyeli veri yapıları veya herhangi bir dosya I/O işlemi yardımı olmadan dinamik boyutların yalnızca işaretçilerle dikkatli bir şekilde ele alınmasına vurgu yapmaktadır. Bellek tahsisi hataları (`malloc`'un `NULL` döndürmesi) durumunda, önceden ayrılmış tüm bellek temizlendikten sonra program sonlandırılmalıdır.
