EN

This problem challenges your mastery of C pointers and dynamic memory management by requiring you to implement a fully dynamic, hierarchical data structure without the aid of `struct` or `typedef`. You will manage a collection of "containers," where each container itself is a dynamic array of integer elements. Both the main collection of containers and the individual containers within it must be able to grow and shrink dynamically based on operations.

Your task is to simulate a rudimentary storage system. You'll need to maintain three parallel dynamic arrays: one `int**` to hold pointers to individual containers, and two `int*` arrays to track the current number of elements (`size`) and the allocated memory (`capacity`) for each corresponding container. Similarly, you must track the total number of containers and the allocated capacity for the main collection itself. This setup forces meticulous manual management of memory blocks and their associated metadata using raw pointers.

You must implement the following operations, ensuring correct memory allocation, deallocation, and resizing without any memory leaks or access violations: 1. **Initialization**: Start with an empty collection. 2. **Add Container**: Dynamically allocate a new container (an `int*` array) and add its pointer to the main `int**` collection, expanding the main collection if necessary. Also, initialize its size and capacity metadata. 3. **Add Element to Container**: Given a container index and an integer value, add the value to the specified container, reallocating the container's memory if its capacity is exceeded. 4. **Remove Element from Container**: Given a container index and an element index, remove the element. This requires shifting subsequent elements and potentially shrinking the container's allocated memory if it becomes sparsely populated. 5. **Remove Container**: Given a container index, free all memory associated with that container, then shift subsequent container pointers and their metadata in the main collection. Potentially shrink the main collection if its capacity is too large relative to the number of active containers. 6. **Display State**: Print the contents, current size, and allocated capacity of all active containers and the main collection. 7. **Cleanup**: Properly free all dynamically allocated memory at the end of the program to prevent leaks. The entire solution must be contained within `main` or use simple helper functions that only interact with primitive types and pointers (no `struct` or `typedef`).

TR

Bu problem, `struct` veya `typedef` kullanmadan tamamen dinamik, hiyerarşik bir veri yapısı uygulamanızı gerektirerek C işaretçileri ve dinamik bellek yönetimi konusundaki ustalığınızı zorlar. Her biri kendi içinde dinamik bir tam sayı elemanları dizisi olan "konteyner" (container) koleksiyonunu yönetecek ve hem ana konteyner koleksiyonu hem de içindeki bireysel konteynerler, işlemlere göre dinamik olarak büyüyüp küçülebilmelidir.

Göreviniz, ilkel bir depolama sistemini simüle etmektir. Üç paralel dinamik dizi tutmanız gerekecektir: biri bireysel konteynerlere işaretçileri tutacak bir `int**` dizisi, diğeri her bir ilgili konteynerin mevcut eleman sayısını (`size`) ve tahsis edilen bellek miktarını (`capacity`) izlemek için iki `int*` dizisi. Benzer şekilde, toplam konteyner sayısını ve ana koleksiyonun tahsis edilen kapasitesini de izlemelisiniz. Bu kurulum, ham işaretçiler kullanarak bellek bloklarının ve ilişkili meta verilerinin titizlikle manuel yönetimini zorunlu kılar.

Bellek sızıntılarını veya erişim ihlallerini önleyerek doğru bellek tahsisi, serbest bırakılması ve yeniden boyutlandırma işlemlerini sağlayarak aşağıdaki işlemleri uygulamanız gerekmektedir: 1. **Başlatma (Initialization)**: Boş bir koleksiyonla başlayın. 2. **Konteyner Ekleme (Add Container)**: Yeni bir konteyneri (bir `int*` dizisi) dinamik olarak tahsis edin ve ana `int**` koleksiyonuna işaretçisini ekleyin, gerekirse ana koleksiyonu genişletin. Ayrıca, boyut ve kapasite meta verilerini başlatın. 3. **Konteynere Eleman Ekleme (Add Element to Container)**: Verilen bir konteyner dizini ve bir tam sayı değeri için, değeri belirtilen konteynere ekleyin, kapasitesi aşılırsa konteynerin belleğini yeniden tahsis edin (`realloc`). 4. **Konteynerden Eleman Çıkarma (Remove Element from Container)**: Verilen bir konteyner dizini ve bir eleman dizini için, elemanı kaldırın. Bu, sonraki elemanları sola kaydırmayı ve konteynerin seyrek hale gelmesi durumunda tahsis edilen belleği potansiyel olarak küçültmeyi gerektirir. 5. **Konteyneri Kaldırma (Remove Container)**: Verilen bir konteyner dizini için, o konteynerle ilişkili tüm belleği serbest bırakın, ardından ana koleksiyondaki sonraki konteyner işaretçilerini ve meta verilerini kaydırın. Ana koleksiyonun kapasitesi, aktif konteyner sayısına göre çok büyükse potansiyel olarak küçültün. 6. **Durumu Görüntüleme (Display State)**: Tüm aktif konteynerlerin ve ana koleksiyonun içeriğini, mevcut boyutunu ve tahsis edilen kapasitesini yazdırın. 7. **Temizleme (Cleanup)**: Programın sonunda tüm dinamik olarak tahsis edilmiş belleği sızıntıları önlemek için doğru bir şekilde serbest bırakın. Tüm çözüm `main` içinde olmalı veya yalnızca ilkel türler ve işaretçilerle etkileşime giren basit yardımcı fonksiyonlar kullanmalıdır (kesinlikle `struct` veya `typedef` yok).
