This problem challenges your understanding of C pointers and dynamic memory management at a very low level. You are tasked with implementing a basic singly linked list. However, there's a crucial constraint: you **must not** use `struct` or `typedef` to define your node structure. This forces you to manage node memory and data access manually using raw pointers and pointer arithmetic.

Each "node" in your linked list should dynamically allocate enough memory to store an single integer value and a pointer to the next node. Specifically, a node should be represented by a contiguous block of memory where the first `sizeof(int)` bytes hold the integer data, and the subsequent `sizeof(void*)` bytes hold the address of the next node. The end of the list is marked by a `NULL` pointer. You will need to carefully cast `void*` pointers to access the integer data and to manipulate the `next` pointer within each allocated memory block.

Your solution must include functions to: 1) `createNode`: allocate memory for a new node, store the given integer value, and initialize its 'next' pointer to `NULL`, returning a `void*` to the newly created node. 2) `insertNode`: create a new node with a given value and insert it at the beginning of the list (updating the head pointer appropriately, which implies using a pointer-to-pointer for the head). 3) `printList`: traverse the list and print all integer elements. 4) `freeList`: deallocate all memory used by the list to prevent memory leaks and set the head pointer to `NULL`. The `main` function should demonstrate these operations by building a small list, printing it, and then freeing it entirely. This exercise rigorously tests your ability to perform low-level memory operations and pointer manipulation without the convenience of higher-level abstractions.

TR

Bu problem, C işaretçileri ve dinamik bellek yönetimi konusundaki anlayışınızı çok düşük bir seviyede zorlamaktadır. Tek yönlü basit bir bağlı liste uygulamanız istenmektedir. Ancak çok önemli bir kısıtlama var: Düğüm yapınızı tanımlamak için `struct` veya `typedef` kullanmamalısınız. Bu durum sizi, düğüm belleğini ve verilere erişimi, ham işaretçiler ve işaretçi aritmetiği kullanarak manuel olarak yönetmeye zorlayacaktır.

Bağlı listenizdeki her "düğüm", bir tek tamsayı değeri ve bir sonraki düğüme işaret eden bir işaretçi saklamak için yeterli belleği dinamik olarak ayırmalıdır. Daha spesifik olarak, bir düğüm, ilk `sizeof(int)` baytın tamsayı verisini tuttuğu ve sonraki `sizeof(void*)` baytın bir sonraki düğümün adresini tuttuğu bitişik bir bellek bloğuyla temsil edilmelidir. Listenin sonu bir `NULL` işaretçisiyle belirtilir. Tamsayı verisine erişmek ve her ayrılmış bellek bloğu içindeki `next` işaretçisini manipüle etmek için `void*` işaretçilerini dikkatlice dönüştürmeniz gerekecektir.

Çözümünüz aşağıdaki işlevleri içermelidir: 1) `createNode`: yeni bir düğüm için bellek ayırır, verilen tamsayı değerini saklar ve 'next' işaretçisini `NULL` olarak başlatır, yeni oluşturulan düğüme `void*` döndürür. 2) `insertNode`: verilen bir değere sahip yeni bir düğüm oluşturur ve onu listenin başına ekler (baş işaretçiyi uygun şekilde günceller, bu da baş için işaretçinin işaretçisini kullanmayı gerektirir). 3) `printList`: listeyi dolaşır ve tüm tamsayı elemanlarını yazdırır. 4) `freeList`: bellek sızıntılarını önlemek için liste tarafından kullanılan tüm belleği serbest bırakır ve baş işaretçiyi `NULL` olarak ayarlar. `main` işlevi, küçük bir liste oluşturarak, onu yazdırarak ve ardından tamamen serbest bırakarak bu işlemleri göstermelidir. Bu alıştırma, yüksek seviyeli soyutlamaların kolaylığı olmadan düşük seviyeli bellek işlemleri ve işaretçi manipülasyonları yapma yeteneğinizi titizlikle test eder.
