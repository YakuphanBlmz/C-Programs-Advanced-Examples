EN

You are tasked with implementing a dynamic, two-dimensional jagged array (matrix) where both the number of rows and the number of columns in each row can grow and shrink independently. Crucially, you must manage all memory allocation and deallocation manually using `malloc`, `realloc`, and `free`. No `struct` or `typedef` are allowed, meaning you cannot bundle row-specific metadata (like current size or capacity) with the row pointer itself. Instead, you must manage these details in separate, dynamically allocated arrays that run parallel to your main `int**` data structure.

Consider a scenario where you need to create such a dynamic jagged array, add several rows, and then add varying numbers of elements to each row. This will necessitate frequent use of `realloc` for both the primary array holding row pointers and for each individual row's array of integers. Additionally, you will need auxiliary dynamic arrays to keep track of the current number of elements and the allocated capacity for each row, as well as the total number of rows and the allocated capacity for the main array of row pointers.

Your solution should demonstrate the creation of such a dynamic jagged array, populating it with some initial data, performing at least one complex operation (such as dynamically resizing a row, inserting an element, or removing an element from a specific position, which involves shifting elements and potentially shrinking memory). Finally, it must completely deallocate all memory allocated throughout the process to prevent any memory leaks. Pay close attention to pointer arithmetic, double pointers (`int**`), and triple pointers (`int***`) if you choose to encapsulate logic within functions that modify the base pointers themselves. Remember, all memory must be freed correctly before program termination.

TR

Hem satır sayısı hem de her bir satırdaki sütun sayısı bağımsız olarak artıp azalabilen dinamik, iki boyutlu, düzensiz (jagged) bir dizi (matris) uygulamanız istenmektedir. `malloc`, `realloc` ve `free` kullanarak tüm bellek tahsis ve serbest bırakma işlemlerini manuel olarak yönetmeniz gerekmektedir. `struct` veya `typedef` kullanılmasına izin verilmediği için, satıra özgü meta verileri (mevcut boyut veya kapasite gibi) satır işaretçisinin kendisiyle birlikte gruplayamazsınız. Bunun yerine, bu ayrıntıları ana `int**` veri yapınıza paralel çalışan ayrı, dinamik olarak tahsis edilmiş dizilerde yönetmelisiniz.

Böyle dinamik düzensiz bir dizi oluşturduğunuz, birden fazla satır eklediğiniz ve ardından her satıra farklı sayıda eleman eklediğiniz bir senaryoyu ele alın. Bu durum, hem satır işaretçilerini tutan birincil dizi hem de her bir satırın tamsayı dizisi için sık sık `realloc` kullanımını gerektirecektir. Ek olarak, her satırın mevcut eleman sayısını ve tahsis edilmiş kapasitesini, ayrıca toplam satır sayısını ve ana satır işaretçileri dizisinin tahsis edilmiş kapasitesini izlemek için yardımcı dinamik dizilere ihtiyacınız olacaktır.

Çözümünüz, böyle dinamik düzensiz bir dizinin oluşturulmasını, bazı başlangıç verileriyle doldurulmasını ve en az bir karmaşık işlemin (bir satırı dinamik olarak yeniden boyutlandırma, bir eleman ekleme veya belirli bir konumdan bir elemanı kaldırma; bu durum elemanların kaydırılmasını ve potansiyel olarak belleğin küçültülmesini içerir) gerçekleştirilmesini göstermelidir. Son olarak, herhangi bir bellek sızıntısını önlemek için işlem boyunca tahsis edilen tüm belleği tamamen serbest bırakmalıdır. Temel işaretçilerin kendilerini değiştiren fonksiyonlar içinde mantığı kapsüllemeyi seçerseniz, işaretçi aritmetiğine, çift işaretçilere (`int**`) ve üçlü işaretçilere (`int***`) özellikle dikkat edin. Unutmayın, program sonlanmadan önce tüm bellek doğru bir şekilde serbest bırakılmalıdır.
