This problem challenges your deepest understanding of C pointers and dynamic memory allocation, pushing you to implement fundamental data structures without the convenience of user-defined types. Your task is to implement a singly linked list where each "node" stores an integer value and a pointer to the next "node". The catch? You are explicitly forbidden from using `struct` or `typedef` keywords. This means you must manually manage the memory layout of each node and perform all data and pointer accesses using raw pointer arithmetic and type casting.

Each "node" in this conceptual list will consist of a contiguous block of memory dynamically allocated using `malloc`. Within this block, the first `sizeof(int)` bytes will store the node's integer data, and the subsequent `sizeof(void*)` bytes will store a pointer to the next "node" block. You will need to carefully calculate memory offsets to read from and write to these locations. The problem requires you to implement several core linked list operations: adding a new node to the end, deleting the first node with a specific value, printing the list, and ultimately freeing all allocated memory.

This exercise is designed to test your mastery of low-level memory management. You will need to think critically about memory alignment, pointer types, and the implications of directly manipulating memory blocks. Success in this problem demonstrates a robust comprehension of how C manages data behind the scenes, far beyond typical usage patterns. Remember to handle edge cases such as an empty list, deleting the head node, and ensuring all dynamically allocated memory is correctly freed to prevent memory leaks. The head of your list will be a `void*` pointer, and you will likely need a `void**` parameter to modify it within functions.

Bu, her düğümün bellek düzenini manuel olarak yönetmeniz ve tüm veri ve işaretçi erişimlerini ham işaretçi aritmetiği ve tür dönüştürme kullanarak gerçekleştirmeniz gerektiği anlamına gelir.
