EN

This challenge requires you to implement a dynamic jagged matrix management system using only raw pointers and dynamic memory allocation functions (`malloc`, `realloc`, `free`). You are tasked with creating and manipulating a two-dimensional array where each 'row' is itself a dynamically sized array of integers, and the number of rows can also change dynamically. Crucially, you are forbidden from using `struct`s, `typedef`s, or file I/O operations; all metadata like row lengths and capacities must be managed through separate, parallel dynamic arrays of integers.

Your implementation must support several key operations. First, you need to initialize the matrix with a specified number of rows, each containing an initial number of integer elements. After initialization, you must be able to add new rows, where each new row is a dynamically allocated integer array of a specified size. Furthermore, the system should allow for resizing existing rows; if a row expands, new elements should be initialized (e.g., to zero), and if it shrinks, excess data is implicitly discarded.

Beyond creation and resizing, your system must also handle the dynamic removal of rows. When a row is removed, its memory must be properly freed, and all subsequent rows (and their associated metadata) must be shifted to maintain data contiguity. Finally, a complete memory cleanup mechanism is essential to free all dynamically allocated resources at the end of the program's execution, preventing memory leaks. This problem heavily tests your understanding of multi-level pointers, dynamic array manipulation, and careful memory management.

TR

Bu problem, sadece ham işaretçiler ve dinamik bellek tahsis fonksiyonları (`malloc`, `realloc`, `free`) kullanarak dinamik, düzensiz bir matris yönetim sistemi uygulamanızı gerektirir. Her "satırın" kendisi dinamik olarak boyutlandırılabilen bir tamsayı dizisi olduğu ve satır sayısının da dinamik olarak değişebileceği iki boyutlu bir dizi oluşturmak ve manipüle etmekle görevlendirileceksiniz. En önemlisi, `struct` yapıları, `typedef` tanımları veya dosya I/O işlemleri kullanmanız kesinlikle yasaktır; satır uzunlukları ve kapasiteleri gibi tüm meta veriler, ayrı, paralel dinamik tamsayı dizileri aracılığıyla yönetilmelidir.

Uygulamanızın birkaç temel işlemi desteklemesi gerekmektedir. Öncelikle, matrisi belirli sayıda satırla, her biri başlangıçta belirli sayıda tamsayı öğesi içeren şekilde başlatmalısınız. Başlatmanın ardından, her yeni satırın belirtilen boyutta dinamik olarak tahsis edilmiş bir tamsayı dizisi olduğu yeni satırlar ekleyebilmeniz gerekmektedir. Ayrıca, sistem mevcut satırların yeniden boyutlandırılmasına izin vermelidir; bir satır genişlerse, yeni öğeler başlatılmalı (örneğin sıfıra), küçülürse ise fazla veriler örtük olarak atılmalıdır.

Oluşturma ve yeniden boyutlandırmanın ötesinde, sisteminiz satırların dinamik olarak kaldırılmasını da yönetmelidir. Bir satır kaldırıldığında, belleği düzgün bir şekilde serbest bırakılmalı ve sonraki tüm satırlar (ve ilgili meta verileri) veri sürekliliğini korumak için kaydırılmalıdır. Son olarak, programın yürütülmesinin sonunda tüm dinamik olarak tahsis edilmiş kaynakları serbest bırakmak ve bellek sızıntılarını önlemek için eksiksiz bir bellek temizleme mekanizması şarttır. Bu problem, çok seviyeli işaretçiler, dinamik dizi manipülasyonu ve dikkatli bellek yönetimini derinlemesine anlamanızı test etmektedir.
