This problem challenges your understanding of advanced pointer manipulation and dynamic memory management in C. You are tasked with implementing a sophisticated filtering mechanism for a dynamically allocated "jagged" 2D array. A jagged array, unlike a standard 2D array, allows each "row" to have a different number of "columns." To adhere strictly to the constraints of this problem, the traditional `struct` keyword is forbidden. Instead, you must devise a clever way to store the length of each dynamic row within the row's allocated memory itself, specifically by making the first element of each row (index 0) store its effective length.

Your primary objective is to develop a C function that can efficiently remove all rows from this jagged 2D array that contain a specified "target value." This operation demands not only careful iteration and conditional freeing of individual row memories but also a comprehensive reallocation of the main `int**` array to reflect the reduced number of rows. The relative order of the remaining rows must be preserved. Furthermore, the overall memory footprint should be minimized by correctly `realloc`ating the main array to precisely fit the new count of valid rows.

The function signature will require a triple pointer `int***` to allow modification of the `int**` array pointer itself (which is held in the caller's scope), and an `int*` to update the count of rows. Proper memory hygiene is paramount; every piece of memory dynamically allocated must be explicitly freed if it is no longer part of the resultant array. This includes freeing individual rows that are discarded and correctly handling the `realloc`ation of the main array pointer to prevent memory leaks and ensure optimal memory usage. Pay close attention to edge cases, such as when all rows are removed or when the initial array is empty.

TR

Bu problem, C programlama dilindeki ileri düzey işaretçi kullanımı ve dinamik bellek yönetimi anlayışınızı zorlayacak. Göreviniz, dinamik olarak ayrılmış "düzensiz" (jagged) bir 2D dizi için gelişmiş bir filtreleme mekanizması uygulamaktır. Standart bir 2D diziden farklı olarak, düzensiz bir dizide her "satır" farklı sayıda "sütuna" sahip olabilir. Bu problemin kısıtlamalarına kesinlikle uymak için geleneksel `struct` anahtar kelimesi yasaktır. Bunun yerine, her dinamik satırın uzunluğunu, o satırın ayrılmış belleğinin içinde, özellikle her satırın ilk öğesinin (0. indeks) etkin uzunluğunu saklamasını sağlayarak akıllıca bir yöntem geliştirmeniz gerekmektedir.

Temel hedefiniz, belirtilen bir "hedef değeri" içeren tüm satırları bu düzensiz 2D diziden etkili bir şekilde kaldıran bir C fonksiyonu geliştirmektir. Bu işlem, yalnızca bireysel satır belleklerinin dikkatli bir şekilde iterasyonunu ve koşullu olarak serbest bırakılmasını değil, aynı zamanda ana `int**` dizisinin, azalan satır sayısını yansıtacak şekilde kapsamlı bir şekilde yeniden tahsis edilmesini de gerektirir. Kalan satırların göreceli sırası korunmalıdır. Ayrıca, ana diziyi yeni geçerli satır sayısına tam olarak uyacak şekilde doğru bir şekilde `realloc` ederek genel bellek ayak izi minimize edilmelidir. 

Fonksiyon imzası, `int**` dizi işaretçisinin kendisinin değiştirilmesine izin vermek için (çağıranın kapsamındaki `int**` işaretçisi), bir üçlü işaretçi `int***` ve satır sayısını güncellemek için bir `int*` gerektirecektir. Uygun bellek hijyeni çok önemlidir; dinamik olarak ayrılmış her bellek parçası, sonuçtaki dizinin bir parçası değilse açıkça serbest bırakılmalıdır. Buna, atılan bireysel satırları serbest bırakmak ve bellek sızıntılarını önlemek ve optimum bellek kullanımını sağlamak için ana dizi işaretçisinin `realloc` işlemini doğru bir şekilde ele almak dahildir. Tüm satırların kaldırılması veya başlangıçtaki dizinin boş olması gibi kenar durumlarına özel dikkat gösterin.
