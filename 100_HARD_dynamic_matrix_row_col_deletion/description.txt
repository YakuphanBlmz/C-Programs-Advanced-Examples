EN

In C programming, managing dynamically allocated two-dimensional arrays, often referred to as matrices, requires careful handling of pointers and memory. A dynamic matrix is typically represented as an `int**`, where the first level pointer points to an array of `int*` (row pointers), and each `int*` then points to an array of `int` (the actual elements of a row). This setup provides great flexibility but also introduces significant complexity in memory management, especially when modifying the matrix's dimensions.

Your task is to implement a robust C function, `deleteRowAndColumn`, which modifies a dynamically allocated integer matrix in-place by removing a specified row and a specified column. The function signature will be `void deleteRowAndColumn(int*** matrix_ptr, int* rows_ptr, int* cols_ptr, int rowToDelete, int colToDelete)`. The `matrix_ptr` parameter is a pointer to the matrix's base pointer (`int**`), allowing the function to update the matrix's address if reallocation occurs or if the matrix becomes empty. The `rows_ptr` and `cols_ptr` parameters are pointers to integers holding the current number of rows and columns, respectively, enabling the function to update these dimensions after the deletion. The `rowToDelete` and `colToDelete` parameters specify the 0-indexed row and column to be removed.

The challenge lies in correctly deallocating the memory for the removed row and the elements of the removed column, and then reallocating and restructuring the remaining elements into a new `(M-1) x (N-1)` matrix. You must ensure that no memory leaks occur, and all dynamically allocated memory for the old structure is properly freed before the new structure is established and referenced by `matrix_ptr`. For simplicity, assume that `rowToDelete` and `colToDelete` will be valid indices within the current dimensions (i.e., `0 <= rowToDelete < *rows_ptr` and `0 <= colToDelete < *cols_ptr`). The solution should also gracefully handle cases where the matrix might become empty (0 rows or 0 columns) after deletion. You are strictly forbidden from using `struct`, `typedef`, or any file I/O operations.

TR

C programlamada, dinamik olarak ayrılmış iki boyutlu dizileri, genellikle matrisler olarak adlandırılan yapıları yönetmek, işaretçiler ve bellek üzerinde dikkatli işlem yapmayı gerektirir. Dinamik bir matris tipik olarak bir `int**` olarak temsil edilir; burada ilk seviye işaretçi bir `int*` dizisine (satır işaretçileri) işaret eder ve her `int*` daha sonra bir `int` dizisine (bir satırın gerçek elemanları) işaret eder. Bu kurulum büyük esneklik sağlar ancak özellikle matrisin boyutlarını değiştirirken bellek yönetiminde önemli bir karmaşıklık getirir.

Göreviniz, belirtilen bir satırı ve belirtilen bir sütunu kaldırarak, dinamik olarak ayrılmış bir tamsayı matrisini yerinde değiştiren sağlam bir C fonksiyonu olan `deleteRowAndColumn`'ı uygulamaktır. Fonksiyonun imzası `void deleteRowAndColumn(int*** matrix_ptr, int* rows_ptr, int* cols_ptr, int rowToDelete, int colToDelete)` olacaktır. `matrix_ptr` parametresi, matrisin temel işaretçisine (`int**`) bir işaretçidir ve yeniden tahsis gerçekleşirse veya matris boş hale gelirse fonksiyonun matrisin adresini güncellemesine olanak tanır. `rows_ptr` ve `cols_ptr` parametreleri, sırasıyla mevcut satır ve sütun sayısını tutan tamsayılara işaretçilerdir ve fonksiyonun silme işleminden sonra bu boyutları güncellemesini sağlar. `rowToDelete` ve `colToDelete` parametreleri, kaldırılacak 0 tabanlı satır ve sütunu belirtir.

Zorluk, kaldırılan satırın ve kaldırılan sütunun elemanlarının belleğini doğru bir şekilde serbest bırakmak ve ardından kalan elemanları yeni bir `(M-1) x (N-1)` matrisine yeniden tahsis edip yapılandırmaktır. Bellek sızıntısı olmamasını sağlamalı ve eski yapı için dinamik olarak ayrılmış tüm belleğin, yeni yapı oluşturulmadan ve `matrix_ptr` tarafından referans alınmadan önce düzgün bir şekilde serbest bırakıldığından emin olmalısınız. Basitlik adına, `rowToDelete` ve `colToDelete`'ın mevcut boyutlar dahilinde geçerli indeksler olacağını varsayın (yani, `0 <= rowToDelete < *rows_ptr` ve `0 <= colToDelete < *cols_ptr`). Çözüm, matrisin silme işleminden sonra boş (0 satır veya 0 sütun) hale gelebileceği durumları da zarifçe ele almalıdır. `struct`, `typedef` veya herhangi bir dosya I/O işlemi kullanmanız kesinlikle yasaktır.
