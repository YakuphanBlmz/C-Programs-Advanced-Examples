The realm of C programming often presents formidable challenges when it comes to low-level memory management, especially when dealing with complex, non-contiguous data structures. This "HARD" problem pushes the boundaries of your understanding of pointers, dynamic memory allocation (`malloc`, `free`), and pointer-to-pointer semantics. You are tasked with meticulously manipulating a dynamically allocated "jagged" 2D array of integers, where each "row" is itself a dynamically sized array. Your goal is to filter this structure based on a numerical property and reconstruct a new, similar structure entirely from scratch, ensuring perfect memory hygiene.

Specifically, you will be provided with an existing dynamically allocated jagged 2D array (represented as an `int**`) and its corresponding dimensions: the total number of rows and an array specifying the number of columns for each individual row. Your primary task is to implement a function that identifies all prime numbers within this original structure. Upon identification, you must construct a *new*, distinct jagged 2D array that contains *only* these prime numbers. The new array must mirror the original row structure where possible; if an original row contains prime numbers, a corresponding row must be created in the new array. If an original row contains no prime numbers, no corresponding row should be created in the new array, effectively pruning empty rows.

The function you implement must return the newly created `int**` structure and update its dimensions (new number of rows, and a new array of column counts for each new row) via pointers. This requires careful handling of `int***` and `int**` parameters. After your function completes, your `main` function must demonstrate proper allocation, population, function call, and crucially, the complete deallocation of *all* dynamically allocated memory for both the original and the newly filtered matrices. Neglecting any `free` operation or performing an invalid `free` will result in memory leaks or runtime errors. This exercise demands precision, a deep understanding of pointer arithmetic, and a robust strategy for dynamic memory lifecycle management.

TR

C programlama dünyası, özellikle karmaşık, bitişik olmayan veri yapılarıyla uğraşırken düşük seviyeli bellek yönetimi konusunda zorlu görevler sunar. Bu "ZOR" seviyedeki problem, işaretçiler, dinamik bellek tahsisi (`malloc`, `free`) ve işaretçinin işaretçisi (pointer-to-pointer) semantiğine dair anlayışınızın sınırlarını zorlayacak. Tamamen dinamik olarak ayrılmış, "düzensiz" (jagged) bir 2D tam sayı dizisini titizlikle manipüle etmeniz isteniyor; burada her "satır" kendi içinde dinamik boyutlu bir dizidir. Amacınız, bu yapıyı sayısal bir özelliğe göre filtrelemek ve sıfırdan tamamen yeni, benzer bir yapı oluştururken kusursuz bellek hijyeni sağlamaktır.

Özel olarak, size mevcut, dinamik olarak ayrılmış düzensiz bir 2D dizi (`int**` olarak temsil edilir) ve ilgili boyutları: toplam satır sayısı ile her bir satırın sütun sayısını belirten bir dizi sağlanacaktır. Temel göreviniz, bu orijinal yapıdaki tüm asal sayıları tanımlayan bir fonksiyon uygulamaktır. Tanımlamadan sonra, yalnızca bu asal sayıları içeren *yeni*, farklı bir düzensiz 2D dizi oluşturmanız gerekmektedir. Yeni dizi, mümkün olduğunca orijinal satır yapısını yansıtmalıdır; eğer orijinal bir satır asal sayılar içeriyorsa, yeni dizide karşılık gelen bir satır oluşturulmalıdır. Eğer orijinal bir satır asal sayı içermiyorsa, yeni dizide karşılık gelen bir satır oluşturulmamalıdır, bu da boş satırların etkili bir şekilde budanmasını sağlar.

Uygulayacağınız fonksiyon, yeni oluşturulan `int**` yapısını döndürmeli ve boyutlarını (yeni satır sayısı ve her yeni satır için yeni sütun sayıları dizisi) işaretçiler aracılığıyla güncellemelidir. Bu durum, `int***` ve `int**` parametrelerinin dikkatli bir şekilde ele alınmasını gerektirir. Fonksiyonunuz tamamlandıktan sonra, `main` fonksiyonunuzun doğru tahsisatı, doldurmayı, fonksiyon çağrısını ve en önemlisi, hem orijinal hem de yeni filtrelenmiş matrisler için *tüm* dinamik olarak ayrılmış belleğin tamamen serbest bırakılmasını (deallocation) göstermesi gerekmektedir. Herhangi bir `free` işlemini göz ardı etmek veya geçersiz bir `free` gerçekleştirmek bellek sızıntılarına veya çalışma zamanı hatalarına yol açacaktır. Bu alıştırma, hassasiyet, işaretçi aritmetiği hakkında derin bir anlayış ve dinamik bellek yaşam döngüsü yönetimi için sağlam bir strateji gerektirir.
