In C programming, managing dynamic multi-dimensional arrays, especially 'jagged arrays' where inner array sizes can vary, is a common challenge that tests a developer's understanding of pointers and dynamic memory allocation. This problem requires you to implement a function that processes a given dynamically allocated jagged array and transforms it based on specific criteria, producing a new dynamically allocated jagged array.

The task is to write a function `void transformJaggedArray(int*** new_jagged_array, int** new_row_sizes, int* new_row_count, int** source_jagged_array, int* source_row_sizes, int source_rows, int sum_threshold)`.

This function will take an existing `source_jagged_array` (which is an array of `int*`, where each `int*` points to a row of integers), its corresponding `source_row_sizes` (an array storing the size of each row), and `source_rows` (the total number of rows). It also receives a `sum_threshold`. The function must dynamically allocate and populate a *new* jagged array according to these rules: 1. Only include rows from the `source_jagged_array` where the sum of its elements is strictly greater than `sum_threshold`. 2. For each selected row, the corresponding row in the `new_jagged_array` should only contain the *even* numbers from the original selected row. All memory for the `new_jagged_array`, `new_row_sizes`, and `new_row_count` must be dynamically allocated *within* the function, and the pointers passed as arguments (`new_jagged_array`, `new_row_sizes`, `new_row_count`) should be updated to point to this newly allocated memory. This requires the use of 'pointer to pointer' concepts for the output parameters.

Your solution must demonstrate robust memory management. All memory allocated for the new jagged array and its associated metadata (row sizes, row count) must be properly allocated. The `main` function should then be responsible for freeing both the original and the newly created jagged arrays, as well as any auxiliary dynamic arrays used (like `source_row_sizes`). Remember, you are forbidden from using `struct`, `typedef`, or `file I/O` operations. Focus on raw pointers and dynamic memory operations.

C programlamada, dinamik çok boyutlu dizileri, özellikle iç dizilerin boyutlarının değişebildiği 'düzensiz dizileri' (jagged arrays) yönetmek, geliştiricinin işaretçiler ve dinamik bellek tahsisi konusundaki anlayışını test eden yaygın bir zorluktur. Bu problem, belirli kriterlere göre dinamik olarak tahsis edilmiş düzensiz bir diziyi işleyen ve dönüştüren bir fonksiyon uygulamanızı gerektirir.

Görev, `void transformJaggedArray(int*** new_jagged_array, int** new_row_sizes, int* new_row_count, int** source_jagged_array, int* source_row_sizes, int source_rows, int sum_threshold)` şeklinde bir fonksiyon yazmaktır.

Bu fonksiyon, mevcut bir `source_jagged_array`'i (her bir `int*`'ın bir tamsayı satırını işaret ettiği bir `int*` dizisi), ilgili `source_row_sizes`'ı (her satırın boyutunu tutan bir dizi) ve `source_rows`'u (toplam satır sayısı) alacaktır. Ayrıca bir `sum_threshold` (toplam eşiği) değeri alır. Fonksiyon, aşağıdaki kurallara göre *yeni* bir düzensiz dizi dinamik olarak tahsis etmeli ve doldurmalıdır: 1. Yalnızca elemanlarının toplamı `sum_threshold`'dan kesinlikle büyük olan satırlar `source_jagged_array`'den yeni diziye dahil edilmelidir. 2. Seçilen her satır için, `new_jagged_array`'deki karşılık gelen satır, orijinal seçilen satırdaki *çift* sayıları içermelidir. `new_jagged_array`, `new_row_sizes` ve `new_row_count` için tüm bellek fonksiyon *içinde* dinamik olarak tahsis edilmeli ve argüman olarak geçirilen işaretçiler (`new_jagged_array`, `new_row_sizes`, `new_row_count`) bu yeni tahsis edilen belleği işaret edecek şekilde güncellenmelidir. Bu, çıktı parametreleri için 'işaretçinin işaretçisi' kavramlarının kullanılmasını gerektirir.

Çözümünüz sağlam bellek yönetimini göstermelidir. Yeni düzensiz dizi ve ilişkili meta veriler (satır boyutları, satır sayısı) için tahsis edilen tüm bellek düzgün bir şekilde tahsis edilmelidir. `main` fonksiyonu, hem orijinal hem de yeni oluşturulan düzensiz diziler için belleği, ayrıca kullanılan herhangi bir yardımcı dinamik diziyi (`source_row_sizes` gibi) serbest bırakmaktan sorumlu olmalıdır. Unutmayın, `struct`, `typedef` veya `dosya işlemleri` (file I/O) kullanmanız yasaktır. Ham işaretçilere ve dinamik bellek işlemlerine odaklanın.
